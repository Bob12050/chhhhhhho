# CLAUDE.md — 開発ガイド（必読の不変ルール）

このリポジトリで作業する全員（人間・AI）が従う規約。新しいシステムを足す前に必ず確認すること。

## 技術スタック

- **ゲームエンジン: Phaser 4.2.0**（Godot ではない。`<技術構成>` 指定により Web ネイティブを採用）
- **言語: 型付き TypeScript 6.0.3**。`any` を避け、`strict` を維持する。
- ビルド: Vite 8 / テスト: Vitest 4 / PWA: vite-plugin-pwa 1 / 保存: IndexedDB
- 依存は `package.json` に**正確なバージョンで固定**し、`package-lock.json` をコミットする。CDN 直読み禁止。
- Phaser の API は**推測で使わない**。`node_modules/phaser/types/phaser.d.ts` と公式 API を確認してから使う（例: Phaser 4 では `setTintFill()` は引数なし。白フラッシュは `setTint(c).setTintMode(Phaser.TintModes.FILL)`）。

## 絶対に守ること

1. **データをハードコードしない。** アイテム/装備/レシピ/職業/スキル/敵/ドロップ/ペット/マップ設定は `src/data/defs/*.json` などのデータで定義する。表示名を参照IDに使わない。全データに不変の文字列IDを持たせる。
2. **ペーパードールのフレーム寸法と基準点を変更しない。** フレーム=**96×96**、足元基準点=**(48,84)**、タイル=32×32。`src/config/resolution.ts` が唯一の真実。変更は全プレースホルダー・全装備レイヤー・全アニメ行に波及する。（2026-06: チビ調の大きめスプライトのため 64×96→96×96 に一度だけ変更済み。以後は固定。）
3. **Nearest フィルタと整数倍率を維持する。** `pixelArt: true` / `roundPixels: true` / `antialias: false`。サブピクセル移動を描画へ反映しない（座標は整数へ丸める）。自由回転・拡大縮小・ぼかしを使わない。
4. **新しいシステムを追加する前に既存構造を確認する。** モジュール責務（`src/` のディレクトリ）に従い、適切な場所へ置く。
5. **巨大クラスを作らない。** `Player` / `GameManager` に全機能を集約しない。責務ごとに分割し、疎結合連携は型付き `EventBus`（`src/core/event-bus.ts`）で行う。文字列イベント名の乱用を避ける。
6. **派生ステータス計算は1か所（`src/stats/stats.ts` の `computeDerived`）に集約する。** UI 表示用と戦闘用で別計算をしない。
7. **ペーパードールはプレイヤーのみ。** 敵・通常NPC・ペットは完成済み単一スプライトを使う。プレイヤーのレイヤーは `PaperDollAnimator` が単一クロックで一括管理し、レイヤーごとに独立タイマーを作らない。
8. **変更後は必ず検証する:** `npm run typecheck` / `npm run validate-data` / `npm run build`（起動確認）/ `npm run test`。
9. **既存作品の素材・固有デザインを模倣しない。** 名称・キャラ・敵・UI・音・画像はオリジナル。
10. **フェーズ外の大規模機能を勝手に追加しない。** Phase 0 が実機で安定するまで Phase 1 のコンテンツを増やさない。各フェーズの完了条件は `docs/ROADMAP.md`。

## モバイル/PWA の遵守事項

- 縦画面専用。横画面ではゲームを停止し警告（`src/scenes/orientation-guard.ts`）。
- safe-area インセットを尊重し、画面下のホームインジケーターに操作ボタンを重ねない。
- セーブデータは **IndexedDB**。Service Worker のキャッシュに入れない。
- 新バージョンは戦闘中に自動再読み込みしない（タイトル復帰後に適用）。
- 画面内通常敵は最大12体程度。ダメージ数字/投射物/ドロップはオブジェクトプール。
- タブ/PWA が非表示になったら一時停止し、自動保存する。

## テストの方針

ゲームロジック（stats / save / drops / leveling / equipment）は **Phaser/DOM 非依存**に保ち、Vitest でヘッドレス実行できるようにする。Phaser に依存するコードはシーン/アクター層に閉じ込める。
