## Phase 0 — 実機検証用の縦切り（現在）

目的: 大量コンテンツ前に、モバイル PWA の基盤が実機で動くことを確認する。

含むもの（実装済み）:
- 縦画面の町テストマップ / プレイヤー移動 / 仮想スティック
- 通常攻撃 / スキルボタン1 / 敵1種 / 素材ドロップ1種
- 頭・胴・武器の装備レイヤー（PaperDollAnimator）/ 装備変更画面
- IndexedDB セーブ・ロード / 自動保存（可視性・装備変更・定期）
- PWA（standalone / portrait / オフライン / 更新通知）
- iPhone/Android の画面比率対応 / 横画面警告 / safe-area

**完了条件（実機）**: 移動→攻撃→スキル→敵撃破→素材ドロップ→装備変更で見た目変化→保存→再起動でロード→オフライン起動→縦横対応が、iPhone Safari/PWA と Android Chrome/PWA でエラーなく動く。

検証コマンド（CI/開発）: `npm run typecheck && npm run validate-data && npm run test && npm run build`。

> Phase 0 が実機で安定するまで、職業・ペット・大量の装備・複数マップを追加しない。
