# GAME_DESIGN.md

## コンセプト

見下ろし型の 2D ドット絵セミオートアクションRPG。プレイヤーの操作テクニックより、**レベル・装備・ステータス・職業・スキル構成による育成と装備構築**を主軸にする。完全オリジナルのモバイル縦画面 PWA。

## ゲームループ

町で準備 → フィールド/ダンジョンへ → 敵を倒す → 素材/装備を入手 → 装備を製作・強化 → レベル/ステータス/スキル/職業/ペットを育成 → 新マップと強敵へ挑戦。

## 操作（モバイル主体）

- 左下: 動的な仮想アナログスティック（触れた位置に出現、離すと中央へ）。
- 右下: 通常攻撃ボタン（長押しで連続）、スキル4枠、状況に応じた「調べる/会話」ボタン。
- 通常攻撃・スキルは現在のロック対象か正面の敵へ軽く自動補正（画面外は対象外）。敵タップでロックオン、空白タップで解除。
- 回避アクションは Phase 1 では実装しない。

## 戦闘フィール

- 4方向移動。右向きは左向き画像の水平反転（左右非対称装備のみ right 専用可）。
- 命中時に短いヒットストップ・ダメージ数字・点滅・ノックバック。
- 敵は予兆攻撃・接触判定・ノックバックを持つ。ボスは予兆範囲攻撃と HP 割合での第2段階。
- ゲーム速度は軽快に、過度なエフェクトは避ける。

## 育成

- 基礎: STR / VIT / INT / DEX / LUK。レベルアップでステータスポイントを獲得し任意割り振り（将来リセット対応）。
- 派生: MaxHP/MaxMP/物理攻撃/魔法攻撃/防御/魔防/命中/回避/会心/攻撃速度/移動速度。基礎値・職業補正・装備・パッシブから計算。
- 職業: tier 0〜4 の分岐ツリー。転職条件はデータ駆動（レベル/現職/クエスト/ボス撃破/素材納品/スキル習得）。
- スキル: 職業別ツリー。アクティブ/パッシブ、スキルレベル、前提、CD、MPコスト等。効果は共通 EffectDefinition の組み合わせ。
- ペット: 追従・経験値/レベル・親密度・支援/攻撃スキル・戦闘不能からの復帰。同時1体（将来は複数/進化/性格/継承）。

## マップ（縦画面設計）

- 進行は上下方向が基本。横に広すぎるマップを作らない。
- 重要な予兆を画面外（特に左右端・画面下＝指で隠れる位置）から出さない。
- NPC/出口は画面上部〜中央に配置。カメラは進行方向を少し先読み。
- ボス部屋は縦長画面で全体を把握できる広さ。

## Phase 1 で遊べる範囲（目標）

町1・フィールド1・ダンジョン1・ボス部屋1、通常敵3+ボス1、初期職+1次職1、アクティブ2+パッシブ1、素材8/装備10〜12/レシピ6/消耗品2、ペット1、簡単なNPC会話、ローカルセーブ、装備による見た目変更、ステータス割り振り、スキル習得、1次職転職。
