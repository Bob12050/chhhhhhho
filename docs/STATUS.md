## 1. システム（機能）の実装状況

### ✅ できている（実装済み・動作確認済み）
- **基盤**: Phaser 4 / TypeScript strict / Vite / Vitest / PWA（standalone・縦画面・オフライン・更新通知）/ safe-area / 横画面警告 / 画面比率対応。
- **操作**: 仮想スティック、攻撃・スキル×2・調べるボタン、キーボード（開発）。
- **戦闘**: 近接攻撃、スキル2スロット＋クールダウン表示、会心、ヒットストップ。
- **戦闘演出**: 斬撃、ヒットスパーク、撃破バースト、被弾の画面赤フラッシュ、低HPビネット、スキル別エフェクト、カメラシェイク、ダメージ数字（プール・ポップ）。
- **敵**: 有限状態機械（待機/徘徊/追跡/攻撃/被弾/帰還/死亡）、単一スプライト、整数スナップ描画。
- **プレイヤー**: ペーパードール（頭・胴・武器レイヤー、単一クロック）。
- **ステータス**: base→derived を `computeDerived` 1か所、レベルアップ、ステ振り。
- **装備**: 9スロット定義、装備/解除で見た目変化、**武器タグの職業ロック**・**防具の系統ロック**・**レア度→職業tierゲート**。
- **レア度**: R1〜R10（色帯は導出）。
- **インベントリ/メニュー**: タブ（素材/消耗/装備/能力/技）、装備のレア度ラベル。
- **クラフト**: 直接生産＋強化（下位装備消費）、ステーション、**武器/防具/どうぐタブ**。
- **ドロップ**: データ駆動テーブル、seedable RNG、ボス初回確定、レアリティ演出（光柱）。
- **経済**: ゴールド、売買（一部）。
- **職業**: 21職ツリー、マルチジョブ育成、転職、解放条件（jobLevel/charLevel/skill/flag/quest）、ツリー閲覧。
- **クエスト**: データ駆動、受注/進捗/達成/報酬、ボードUI、職業解放フラグ連携、看板モンスター構造。
- **マップ**: 5種（町/草原/ささやきの森/洞窟/ボスの間）、ポータル、ファストトラベル、NPC。
- **会話**: ダイアログ＋フラグ設定。
- **セーブ**: IndexedDB、バージョン管理、defensive migrate。
- **デバッグ**: メニュー、ワープ（リリース無効化）。

### 🟡 仕組みはあるが中身が薄い
- **スキル**: 3つだけ（斬撃・強打・頑強）。職業別スキルツリーは未。
- **ペット**: 1体だけ。進化・合成・性格・継承は未。
- **会話/NPC**: 数体・数本のみ。
- **売買/店**: 売却中心、本格的な店UIは簡易。

### ❌ できていない（未実装）
- **エンドゲーム（item_system_spec の調査クエスト系）**: 調査素材、R9/R10固定装備、調査ランダム装備（テンプレ/生成/ドロップ/強化）、分解、ItemInstance のセーブ対応。
- **状態異常・属性**: `element` フィールドはあるが未使用。
- **効果音・音楽**: 無し。
- **防具の部位**: 手/腰/足/背、装飾品（スロットはあるがアイテム0）。
- **tier2〜4 の実コンテンツ**: ゾーン/敵/素材供給が薄い（装備の枠は R1〜R8 まで存在するが供給が追いついていない、下記ギャップ）。
