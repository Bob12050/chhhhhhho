# CONTENT_MAP.md — 進行マップ（tier × レア度 × コンテンツ）

本作の進行の背骨。**職業の段階（tier）とレア度（R）と入手ゾーンを一直線に対応させる**。
データは全部 JSON（`src/data/defs/`）。絵は `visualId` 経由で**後差し**できる（`DATA_SCHEMA.md`）。

> 数量はデータ駆動で後から足せる。まず各 tier を「遊べる密度」にし、絵は並行して差し込む。
> 不変ルールは `CLAUDE.md` 優先。フェーズ順（`ROADMAP.md`）を守る。

---

## 1. 背骨：tier ↔ レア度 ↔ 職業段階

| tier | 職業段階 | レア度帯 | 解放 | 位置づけ |
|---|---|---|---|---|
| 0 | 冒険者 | **R1** | 開始時 | チュートリアル帯 |
| 1 | 1次職 | **R2〜R3** | 冒険者Lv20で転職 | 序盤 |
| 2 | 2次職 | **R4〜R6** | 1次職育成→転職 | 中盤 |
| 3 | 3次職 | **R7〜R8** | 2次職育成→転職 | 終盤（本編クリア） |
| 4 | 4次職 | **R8〜R10** | クリア後＋高難度クエスト | エンドゲーム（調査） |

R8 は tier3/tier4 の重なり（「R band → 必要 tier 以上」で成立）。

## 2. 装備の tier ゲート（要実装・小）

「レア度 R を装備するには職業 tier が一定以上」という**派生ルール**で進行を固定する：

```
R1 → tier0以上 / R2-3 → tier1以上 / R4-6 → tier2以上 / R7-8 → tier3以上 / R9-10 → tier4以上
```

- 実装：`canEquip` にレア度→必要tier判定を追加（職業データの `tier` を参照）。アイテム毎の追加データ不要。
- 既存の `levelRequirement`（数値）は**補助**として残す（tier ゲートが主、レベルは下限の目安）。
- 装備不可時は UI に「職業段階が必要」を表示（既存の「職業不可」と同様）。

> 未確定: levelRequirement を完全廃止するか。当面は残置（無害）。

## 3. 武器タイプ × 職業系統（実装済み）

| 系統(family) | 1次→4次職 | 武器タグ |
|---|---|---|
| warrior | ファイター→サムライ→ソードカイザー→アラミカグラ | sword, axe, spear, katana |
| mage | メイジ→ソーサラー→グランマギアー→アルヴライド | staff, wand |
| cleric | プリースト→ホーリーナイト→シルドセイバー→ニルバディオ | mace, staff, sword, shield |
| thief | シーフ→ニンジャ→アベンジスタ→ノクスティア | dagger, sword, shuriken |
| tamer | ペットライザー→レンジャー→デュアルスター→オルタリエ | whip, wand, bow |

各 tier では、**全 family がその帯の武器・防具を持てる**ように装備を用意する（family ごとに最低1武器タイプ）。

## 4. 見た目（プレースホルダー）方針

絵が揃うまでは既存6種を流用する。データは正式に作り、絵だけ後で差す。

| 種別 | 暫定 visualId | 本番（後差し） |
|---|---|---|
| 全武器（剣/斧/杖/弓…） | `sword_wood`(低) / `sword_iron`(高) | タイプ別スプライト |
| 頭防具 | `cap_leather` / `helm_iron` | 部位別 |
| 胴防具 | `vest_cloth` / `plate_iron` | 部位別 |
| 手/腰/足/背 | （胴/頭を流用） | 部位別 |

絵ができたら `visual-ids.ts` ＋ `visuals.ts` に登録し、該当アイテムの `visualId` を差し替えるだけ（データ無変更）。

## 5. tier 別コンテンツ計画（現状 → 目標密度）

「目標」は各 tier を遊べる最小密度の目安。多すぎたら分割投入。

### tier 0 — 冒険者 / R1
- ゾーン: 町・最初の野原（現状: town, field）
- 敵: スライム, コウモリ（現状あり）
- 素材: slime_jelly, soft_leather, herb（R1, 現状あり）
- 装備: R1 各 family のスターター武器＋頭/胴（現状: 剣/頭/胴のみ → 他 family の R1 武器を追加）

### tier 1 — 1次職 / R2〜R3
- ゾーン: 草原奥・森・洞窟入口（目標 +2〜3）
- 敵: 目標 4〜5種（R2〜R3素材を落とす）
- 素材: iron_ore, mana_stone(R3) ＋ R2帯を1〜2種追加
- 装備: R2〜R3 を全 family 分（武器＋頭/胴）。直接生産＋強化レシピ
- ボス: 1次職トライアル 1体

### tier 2 — 2次職 / R4〜R6
- ゾーン: ダンジョン深部・山・遺跡（目標 +3）
- 敵: 目標 5〜6種
- 素材: steel_ingot(R4), king_jelly(R5), mythril_ore(R6) ＋補強
- 装備: R4〜R6 を全 family 分
- ボス: 2次職トライアル

### tier 3 — 3次職 / R7〜R8
- ゾーン: 魔城・火山 等（目標 +3）
- 敵: 目標 5〜6種（強）
- 素材: golem_core(R7), shadow_wing(R7), star_fragment(R8) ＋補強
- 装備: R7〜R8 を全 family 分
- ボス: **ラストボス（本編クリア地点）**

### tier 4 — 4次職 / R8〜R10（クリア後・調査）
- `item_system_spec v0.1` の調査クエスト系（ギルクエ風）。R9固定＝R8＋調査素材、R10固定＝R9固定＋調査素材、調査ランダム装備（武器/防具のみ）。
- 素材: 調査証・ボスコア（R9/R10）。
- セーブの ItemInstance 対応が前提（最後に着手）。

## 6. 供給ギャップ（即対応候補）

| 素材 | レア度 | 状況 | 方針案 |
|---|---|---|---|
| steel_ingot | R4 | ドロップ源なし | 製錬レシピ（iron_ore→steel_ingot） |
| mythril_ore | R6 | ドロップ源なし | ドロップ（ゴーレム低率＋石像ボス） |
| golem_core / star_fragment | R7/R8 | 率が渋い・敵1体のみ | tier3 の敵/ゾーン拡充で供給を作る |

## 7. 生産の進め方（黄金ループ・`CONTENT_PLAN.md` と併用）

tier 単位で「敵/ゾーン → 素材ドロップ → 装備＋レシピ」を1帯ずつ作り、毎回
`validate-data → typecheck → test → build` で検証。実機は `?debug=1`。
1帯が遊べる密度になったら次 tier へ。tier4（調査）は最後。
