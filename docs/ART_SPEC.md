# ART_SPEC.md

ドット絵の一貫性を最優先する。数値は `src/config/resolution.ts` と `src/paperdoll/pose-atlas.ts` が唯一の真実で、勝手に変更しない。

## 解像度・スケーリング

- 論理幅: **360 固定**（横方向に拡張しない）。
- 論理高さ: 端末アスペクト比に応じて **640〜800** に可変（高さが余る端末は上下の表示範囲を増やす）。
- 整数倍率で拡大し、端数はレターボックス（余白）。Nearest Neighbor、アンチエイリアス無効。
- カメラ・スプライト座標は整数ピクセルへ丸める。サブピクセル移動を描画へ反映しない。自由回転/拡大縮小/ぼかし禁止。

## サイズ・基準点

- タイル: **32×32**。
- キャラクター1フレーム: **64×96**（本体は約52px高 ≒2タイル、上に約40pxの頭上余白＝大きめの帽子/兜用）。
- 足元基準点: **x=32, y=84**（枠中心が x=32 なので水平反転しても整合する）。
- 透明背景・1ピクセル輪郭・色ごと3〜4段階の陰影（outline/shadow/mid/light）。
- 頭が大きく身体が小さい、ゆるく可愛いデフォルメ。背景よりキャラ/敵のコントラストを強く。

## ポーズアトラス（全キャラ・全装備レイヤー共通）

行 = 方向 × アニメーション、列 = フレーム。方向は **down / up / left** をシートに持ち、**right は left の水平反転**（左右非対称装備のみ right 専用可）。

| アニメ | フレーム | ループ |
|---|---|---|
| idle | 2 | ○ |
| walk | 4 | ○ |
| attack | 4 | × |
| cast | 4 | × |
| hurt | 2 | × |
| death | 4 | × |

- シート寸法 = `MAX_FRAMES(=4)×64` 幅 × `(3方向×6アニメ=18行)×96` 高。フレーム範囲はコードに散在させず `pose-atlas.ts` に集約。

## レイヤー（描画グループ・プレイヤーのみ）

shadow / behind_body / base_body / feet / waist / torso / far_hand / far_weapon / back / head / near_hand / near_weapon / front_accessory / front_effect。向きによって武器・手・背中の前後順を入れ替える（`config/layers.ts`）。

1つの装備ビジュアルは behind/main/front シート・方向別表示/反転可否・色違いパレット・対応体型・`visual_family_id` を持てる設計。色違いはパレット差し替えか事前生成画像で表現し、装備品ごとに別画像を要求しない（例: 木の剣/鉄の剣/炎の剣が同一シルエットを共有）。

## プレースホルダー

最終アートを自動生成しない。Phase 0 はコード生成のプレースホルダー（`src/assets/gen/`）を使い、輪郭・パレット・フレーム寸法・基準点・アニメ配置を最終仕様と同一にする。エフェクトも内部解像度上で描画しドット感を保つ。
